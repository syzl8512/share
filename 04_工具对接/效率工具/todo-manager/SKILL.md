# Todo Manager - 待办管理 Skill

> 自动将用户口述的待办进行分类、整理、同步

## 触发条件

当用户发送包含以下关键词时：
- "待办"、"TODO"、"还有什么事"
- "帮我整理一下"、"分类一下"
- 新增任务但未分类

## 执行流程

### 1. 读取当前待办

```
读取 memory/todo.md
```

### 2. 智能分类

根据任务内容自动归类：

| 分类 | 关键词 | 示例 |
|------|--------|------|
| 🖥️ 系统问题 | 浏览器、邮箱、模型、Obsidian、部署 | 解决浏览器问题、邮箱下载 |
| 💪 能力卡点 | 学习、重构、创作、优化 | mia_vault重构、书籍创作 |
| 💼 工作 | 客户、拜访、PPT、报销、项目 | AI企业合作、火车票报销 |
| 🏠 生活 | 家庭、孩子、购物、旅行 | 带孩子去公园 |

### 3. 去重与合并

- 检查是否有重复任务
- 相似任务合并
- 保留最新表述

### 4. 更新文件

更新 memory/todo.md，格式：

```markdown
# 待办清单（分类整理）

> 最后更新: YYYY-MM-DD

---

## 🖥️ 系统问题

- [ ] 任务1
- [ ] 任务2

## 💪 能力卡点

- [ ] 任务1

## 💼 工作待办

- [ ] 任务1

## 🏠 生活待办

- [ ] 任务1

---

## 📅 今日专注

**今日TOP 3:**
1.
2.
3.
```

### 5. Git 同步

```bash
cd ~/.openclaw/workspace
git add memory/todo.md
git commit -m "update: 待办清单更新"
git push
```

### 6. 输出给用户

- 列出更新后的分类
- 询问用户想先推进哪项
- 给出建议

---

## 运转规则（可配置）

| 任务 | 时间 | 方式 |
|------|------|------|
| 早会 | 8:30 | 询问当日计划 |
| 提醒 | 每2小时 | 建议+选择 |
| 总结 | 23:00 | 当日总结 |

---

## 新增定时任务模板

如需添加定时任务，使用 cron.add：

**早会任务：**
- schedule: 30 8 * * *
- payload: 询问计划 + 昨日总结

**工作提醒：**
- schedule: 0 9-17/2 * * *
- payload: 选择2-3项适合当前时段的任务，建议推进

**每日总结：**
- schedule: 0 23 * * *
- payload: 总结今日进展 + 列出待推进事项

---

## 注意事项

1. 不要命令用户，用建议+选择的方式
2. 保持分类简洁，不要过于复杂
3. 定期清理已完成任务
4. 重要任务同步到 GitHub
5. 今日专注每天更新
